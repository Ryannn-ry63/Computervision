这是一个课程汇报项目，主题是“运动物体检测：传感器、方法、挑战及发展趋势”。当前 PPT 的第三部分是“多传感器融合”，其中第 7 页讲的是融合前提：时间同步、空间标定、运动补偿。希望在这一页加入一个真实可视化复现，直观展示相机与 LiDAR / 雷达如何对齐。

汇报时间较短，复现目标不是训练新模型，而是产出可以放进 PPT 的高质量可视化图片或视频。优先级是：效果直观、能解释工作量、能和“空间标定/BEV融合”内容对应。

1. 主复现：尝试使用 MMDetection3D 中的 BEVFusion 或多模态 3D 检测 demo，在 nuScenes 样例上跑出相机 + LiDAR 融合检测可视化。

## 推荐路线 B：真实 KITTI / nuScenes 点云投影可视化

这是保底路线，也非常适合第 7 页。它不需要训练模型，不依赖大规模权重，只需要一帧相机图像、LiDAR 点云和标定文件。

目标：读取 KITTI 标定参数，把 LiDAR 点云投影到相机图像，按深度着色，并额外输出 BEV 鸟瞰图。

### 1. 数据来源

任选一种：

- 使用 KITTI object detection 的一个样例：`image_2/*.png`、`velodyne/*.bin`、`calib/*.txt`
- 使用 MMDetection3D 自带 demo 数据：`demo/data/kitti/000008.bin` 等，如果有对应图像和标定文件
- 使用 nuScenes mini 样例，通过 nuScenes devkit 渲染 `render_pointcloud_in_image`

KITTI 最简单，优先 KITTI。

### 2. 投影公式

KITTI 标定文件中通常有：

- `P2`：左相机投影矩阵，3x4
- `R0_rect`：校正矩阵，3x3
- `Tr_velo_to_cam`：LiDAR 到相机坐标系外参，3x4

将 LiDAR 点 `X_velo = [x, y, z, 1]^T` 投影到图像：

```text
X_cam = R0_rect * Tr_velo_to_cam * X_velo
u_v_z = P2 * X_cam
u = u_v_z[0] / u_v_z[2]
v = u_v_z[1] / u_v_z[2]
```

只保留：

- `X_cam.z > 0`
- `0 <= u < image_width`
- `0 <= v < image_height`

颜色用深度 `X_cam.z` 映射，例如 `matplotlib` 的 `jet` 或 `turbo` colormap。

### 3. 期望脚本功能

请写一个脚本，例如 `scripts/kitti_lidar_projection.py`，支持：

```bash
python scripts/kitti_lidar_projection.py \
  --image path/to/image_2/000000.png \
  --pointcloud path/to/velodyne/000000.bin \
  --calib path/to/calib/000000.txt \
  --out-dir outputs/fusion_reproduction
```

输出：

- `camera_raw.png`
- `lidar_projection_depth.png`
- `bev_pointcloud.png`

### 4. 成功后汇报口径

如果只跑通 KITTI 投影，第 7 页可以这样讲：

> 这张图是我用 KITTI 数据集的一帧样本复现的结果。左侧是原始相机图像，中间是利用标定文件将 LiDAR 三维点云投影到相机图像后的结果，颜色表示深度；右侧是同一帧点云的 BEV 鸟瞰图。这个实验展示了多传感器融合最基础的一步：不同传感器的数据必须先通过外参和内参变换对齐到同一坐标系，后续的 PointPainting、BEVFusion 等方法才能在此基础上进行融合。

## 不建议优先做的路线

### 1. 完整训练 BEVFusion

不建议为了课程汇报训练 BEVFusion。数据量大、训练时间长、版本依赖重。使用预训练权重跑 demo 即可。

### 2. 完整复现 CLOCs

CLOCs 需要 2D 检测器和 3D 检测器的候选框结果，再做候选框融合。它适合论文讲解，不适合短时间做主展示。

### 3. 完整 K-Radar 训练

K-Radar 数据和代码适合讲 4D 雷达趋势，但完整数据体量非常大。可以参考其可视化思想，不建议把完整 K-Radar 训练作为主目标。
